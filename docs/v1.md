# Chalagente v1

Next.js deployed to Cloudflare.

## v1.1
We're going to use WhatsMeow for the WhatsApp interface, just as a proof of concept.

Therefore we need a flow where:
- we (the 'business') scan a QR code generated by WhatsMeow
- that creates a 'linked device', for which we store the credentials
- the 'customer' sends a message to the number associated with the linked device
- the 'customer' receives a hardcoded response

Given that the frontend is a Next.js app, either we need to nuke the Next app and just have a single Go app, or we need a monorepo where the WhatsMeow and frontend communicate over HTTP. For simplicity let's go for a single Go app that wraps WhatsMeow.
